# ETF 多因子策略 —— 券商模拟盘操作 SOP

> 目标:验证①信号能否按月稳定产出 ②真实成交 vs 回测口径 ③业绩是否跟得上回测。
> 频率:**每月末做一次**(月度调仓)。全程手动,约 30 分钟。

---

## 一、首次准备(只做一次)

1. **开模拟交易账户**:在常用券商 App(华泰涨乐/国君/东财等)开"模拟交易"账户,初始资金建议 **100 万**(对应回测里小资金甜区,lam 可保持 0.4;若想测更小资金可调高 lam 到 0.6)。
2. **建一个跟踪表** `paper_log.csv`,每月记一行:日期 / 总资产 / 当月收益 / 对应回测同期收益。用于目标③业绩比对。
3. 记住纪律(报告结论):**固定参数永不优化、月度调仓、牛市跑输不动摇**。这本身就是要演练的一部分。

---

## 二、每月流程(月末最后一个交易日 T 日收盘后)

### 第 1 步 · 刷新行情数据
打开 iFinD,运行 `ifind_etf_history.ipynb` 把 `etf_market_ifind.db` 更新到 T 日收盘。
> 验证目标①:若某月数据拉不全/有缺口,就是这一步暴露,记录下来。

### 第 2 步 · 生成目标权重
```bash
python3 -m etf_factor_strategy.cli --end <T日,如 2026-06-30>
```
产出 `outputs_v2_final/rebalance_weights.csv`(最新调仓日那 20 行就是本月目标)。

### 第 3 步 · 生成下单清单
```bash
# 首月建仓(一次铺满目标):
python3 live_orders.py --capital 1000000 --first

# 此后每月(先把上月成交后的真实持仓填进 holdings.csv):
python3 live_orders.py --holdings holdings.csv
```
`holdings.csv` 格式(从 App 抄每只的"份额",现金单独一行):
```
fund_code,shares
511880,1200
159972,1000
...
CASH,30000
```
输出 `trade_orders.csv` + 屏幕清单,含:**方向 / 买卖手数 / 约需金额 / 折溢价% / 近20日均额 / 红线提示**。
> 下单助手已内置 lam=0.4 部分再平衡(`新持仓=当前+0.4×(目标−当前)`,与回测算法一致)。

### 第 4 步 · 处理红线(验证目标②的关键)
对带 `⚠` 的标的:
- **折溢价大(>2%,主要是 QDII)**:下单**限价贴 IOPV**,溢价过高就**减半或跳过**,记录跳过原因。
- **流动性低(近20日均额<3000万)**:按"单笔≤当日成交额一定比例"分批,或跳过换下一名。
> 这一步的偏差(跳过了哪些、成交价比月末收盘差多少)就是回测 5bps 成本假设是否成立的实测证据。

### 第 5 步 · T+1 下单
**T+1 尾盘 30 分钟(约 14:30–14:57)分批挂限价单**(最贴近回测的收盘口径,避开开盘跳空)。**不要开盘下单**。

### 第 6 步 · 记录
成交后:
- 把真实成交份额回填进 `holdings.csv`(供下月用);
- 在 `paper_log.csv` 记当月总资产;
- 记下"成交均价 vs 当日收盘价"的偏差、跳过的标的。

---

## 三、跑满后怎么判断"能不能上真钱"

跑 **6–12 个月**(至少覆盖一次明显的涨/跌切换),看:

| 检查项 | 通过标准 |
|---|---|
| 信号稳定性(目标①) | 每月都能按时产出,无频繁数据缺口 |
| 成交一致性(目标②) | 实际成交价 vs 月末收盘的滑点 + 跳过标的的拖累,**合计 ≤ 回测假设的 5bps/边量级**;否则要上调成本重估 |
| 业绩跟踪(目标③) | 模拟净值曲线与回测同期**方向一致、偏离可解释**(防御年抗跌、牛市跑输属正常,不是失效) |
| 纪律 | 中途没有手痒改参数/择时 |

三项都过 → 按报告容量曲线,真钱从小仓起步、逐步放大至 ≤3 亿。任一项不过 → 留在模拟盘继续。

---

## 四、注意事项

- **首月口径差异**:`--first` 是一次性铺满目标;回测是按 lam 每月 40% 渐进建仓(前几个月仓位偏低)。两者前 ~5 个月会有差异,之后收敛一致。比对业绩时对首月单独标注即可。
- **现金缓冲**:净买入金额可能略超总资产(整百取整所致),实际下单留 1–2% 现金。
- **`fund_hit_rate_20` 因子**:报告 §8 建议降权/剔除。若想在模拟期就用更纯的版本,改 `engine.py: FACTOR_WEIGHTS_V2` 后重测再上(否则保持现状,模拟盘也跑现状版)。
