# A股多因子 Alpha 研究框架 — 设计文档

> 目标：一个用于量化私募实习面试的作品集项目。同时展示工程力、研究规范、研究品味、ML 方法论与"诚实"。
> 日期：2026-06-26

## 一、项目定位

不追求"高夏普神因子"，而是证明四件量化私募真正筛选的能力：
1. **研究规范 / 不作弊**：无前视、处理 ST/次新/停牌/涨跌停/退市、扣成本与换手。
2. **工程细节**：因子衰减、容量、拥挤度、换手-成本权衡。
3. **科学方法**：先有经济假设再验证，而非暴力搜参。
4. **诚实**：敢说哪些不work、为什么，量化容量上限。

## 二、数据底座（已就绪）

- `mart/fact_stock_daily.parquet`：5896 只 A 股、2011–2026、1365 万行日频。含 `adj_close`、`total/float_mktcap`、`pe_ttm/pb/ps_ttm`、`turnover`、`is_st`、`limit_status`。
- `mart/fact_stock_factors.parquet`：附 beta、特质波动（`nonsys_risk`）、走势特征（`up_days`/`is_life_high` 等）。
- `mart/fact_ff3_daily` / `fact_ff5_daily`：FF 因子；`TRD_Cndalym` 市场收益；`TRD_Nrrate` 无风险利率；`IDX_Idxtrd` 指数（沪深300/中证500）。

### 关键数据真相（已核对，决定回测正确性）
- **停牌 = 当天无此行**（全表无 volume=0）。股票池须 point-in-time 动态生成。
- **`ret` 是含分红的总收益**；多日动量用 `adj_close`，勿用原始 `close`。
- **退市股保留**（293 只 last_trddt<今），无幸存者偏差。
- `limit_status`：-1 跌停 / 0 正常 / 1 涨停。
- **缺口**：无行业分类表 → v1 仅做市值中性化，行业做成可插拔接口，待补数据。

## 三、架构

```
quantlib/                 # 可复用引擎（项目的"产品"）
  data.py                 # 从 mart 取 point-in-time 面板（DuckDB → pandas）
  universe.py             # 股票池：剔除 ST/次新(<120日)/停牌/当日涨跌停不可交易
  preprocess.py           # 去极值(MAD) → 中性化(市值，行业可插) → 标准化(zscore)
  evaluate.py             # IC/RankIC/ICIR/t值、分层回测、IC衰减
  backtest.py             # 组合回测：约束、换手、交易成本、对标基准
  factors/                # 每个因子 = 一个函数 f(panel)->Series
configs/                  # 频率、成本、股票池参数
research/                 # 分层分析报告（notebook）
tests/                    # 防泄漏单元测试
results/                  # 图与表
```

设计原则：因子是纯函数 `f(panel)->Series`，引擎对所有因子一视同仁；加新因子的边际成本≈写一个函数。

## 五、回测设定

- **股票池**：全 A 股；剔除 ST、次新(<120 交易日)、调仓日停牌、当日涨跌停不可成交；退市股保留。
- **调仓频率**：参数化，跑**月频(baseline)与周频(进阶)**，对比成本敏感性。
- **收益**：用 `ret`（含分红）；多日信号用 `adj_close`。
- **成本**：双边千分之三（含佣金+印花税+冲击）可配；周频下重点分析。
- **基准**：沪深300、中证500。
- **评价**：IC/RankIC/ICIR、分层多空、年化超额、IR、最大回撤、换手率、容量估算。

## 六、明确不做（YAGNI）

- 不做日内/高频。
- 不做实盘对接。
- 不做组合优化器的复杂版（先用简单加权 + 线性约束，够讲清楚即可）。
- 行业中性化 v1 暂缓（缺数据），接口预留。
