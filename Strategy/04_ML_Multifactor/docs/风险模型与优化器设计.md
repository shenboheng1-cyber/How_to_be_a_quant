# 风险模型 + 组合优化器 设计（L5）

> 补项目最大短板：从 L0 起只做了市值中性、组合是 naive top-decile 等权。
> 目标：Barra 式风险模型 + 约束优化组合（行业/风格中性、换手/权重/跟踪误差约束）。

## 第一步：下载行业分类（必需）
| 数据库 | 表 | 用途 |
|---|---|---|
| 上市公司基本信息 | `STK_INDUSTRYCLASS`（上市公司行业分类表） | 个股→行业映射（含变更日期，**PIT 用生效日**） |
| 上市公司基本信息 | `STK_INDCLASSIFYSETS` / `STK_INDCLASSIFYVERSION` | 选分类标准 |

**用哪个标准**：推荐**申万一级(~31行业)** 或 **证监会2012(~19门类/90大类)**。一级 ~31 个行业做哑变量正合适。
**PIT 坑**：行业分类会变更，`STK_INDUSTRYCLASS` 带生效/变更日期，回测每期用当时有效的分类，不能用最新快照。

## 第二步：Barra 式多因子风险模型
**因子暴露 X**（每股每期）：
- 行业暴露：~31 个行业哑变量(0/1)。
- 风格暴露：你已有的因子标准化值——Size、Value(BP/EP)、Momentum、Volatility、Liquidity、Growth、Leverage 等（约8-10个大类风格）。

**估计**（每个调仓期横截面回归）：
```
r_i = Σ_industry f_ind·X_ind + Σ_style f_sty·X_sty + ε_i
```
- 得到因子收益 f（时序）→ 因子协方差矩阵 **F**（用历史 f 估，可加 EWMA/Newey-West）。
- 残差 ε 的方差 → 特质风险 **Δ**（对角）。
- 组合风险：**σ²(w) = wᵀ(XFXᵀ + Δ)w**。

## 第三步：组合优化器（二次规划）
```
max_w   α·w  −  λ·wᵀ(XFXᵀ+Δ)w        # alpha=ML合成信号
s.t.    Σw = 1
        Xᵀ_industry·w = b_industry     # 行业中性(对标基准权重)
        |Xᵀ_style·w| ≤ s               # 风格暴露上限(如市值中性)
        0 ≤ w ≤ w_cap                  # 个股权重上限(如3%)
        Σ|w − w_prev| ≤ turnover_cap   # 换手约束
```
- 求解：`cvxpy`(QP)；或先做"行业中性 + 市值中性 + 权重上限"的简化版（线性约束下的排序加权）。
- 输出：每期最优权重 w → 替换 backtest 里的 top-decile 等权。

## 接入现有引擎
- `quantlib/riskmodel.py`：估因子收益/协方差/特质风险。
- `quantlib/optimizer.py`：QP 求解每期权重。
- `backtest.py` 增加 `weights=` 入口，接受优化器权重而非等权。

## 依赖
`cvxpy`（QP 求解）。先验证 csmar/base 是否可用，否则用 `scipy.optimize` 或闭式近似。

## 预期收益
- **行业/风格中性** → 超额更纯(不是行业 beta)、IR 提升、回撤降。
- **换手约束** → 扣成本后净 IR 提升、容量上升。
- 这是"学生项目→专业系统"最关键一跃，面试最加分。
